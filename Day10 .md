## Topics Covered

### 1. Introduction to System Logs

A **log** is a record of events generated by an operating system, application, or network device. Every activity performed on a computer system, such as user logins, software execution, system startup, or errors, can be recorded in log files. These logs help administrators and cybersecurity professionals understand what is happening on a system.

---

### 2. Importance of Logs

The instructor explained that logs play a critical role in cybersecurity because they help to:

- Monitor system and network activities.
- Detect unauthorized access attempts.
- Identify suspicious or malicious behavior.
- Troubleshoot software and hardware issues.
- Investigate cyber incidents.
- Perform digital forensic investigations.
- Maintain compliance with security standards and organizational policies.

---

### 3. Types of Logs

We studied several important categories of logs:

#### System Logs
- Record operating system events such as system startup, shutdown, hardware changes, and kernel activities.

#### Authentication Logs
- Store successful and failed login attempts.
- Help identify unauthorized access and brute-force attacks.

#### Application Logs
- Record events generated by installed software and applications.
- Useful for debugging and troubleshooting application issues.

#### Security Logs
- Contain security-related events such as privilege changes, firewall alerts, and policy violations.

#### Network Logs
- Record incoming and outgoing network traffic.
- Help identify suspicious connections, scanning attempts, and network attacks.

---

### 4. Log Monitoring

The instructor explained that security analysts continuously monitor log files to:

- Detect unusual activities.
- Identify failed login attempts.
- Discover malware infections.
- Monitor system health.
- Respond quickly to security incidents.

---

### 5. Log Analysis

Log analysis is the process of examining log files to:

- Detect suspicious patterns.
- Investigate cyberattacks.
- Identify the root cause of problems.
- Improve the overall security posture of an organization.

## Commands Practiced

### View complete system logs
```bash
sudo journalctl
```
**Purpose:** Displays all system logs stored by the systemd journal.

### View logs from the current boot
```bash
sudo journalctl -b
```
**Purpose:** Shows logs generated since the last system startup.

### View the latest 50 log entries
```bash
sudo journalctl -n 50
```
**Purpose:** Displays the most recent 50 log records.

### Monitor logs in real time
```bash
sudo journalctl -f
```
**Purpose:** Continuously displays new log entries as they are generated.

### View kernel log messages
```bash
dmesg
```
**Purpose:** Displays kernel and hardware-related messages.

### View authentication logs
```bash
cat /var/log/auth.log
```
**Purpose:** Displays user authentication and login records.

### Monitor authentication logs
```bash
tail -f /var/log/auth.log
```
**Purpose:** Displays new authentication log entries in real time.

### View the last 20 lines of system logs
```bash
tail -20 /var/log/syslog
```
**Purpose:** Shows the latest entries from the system log file.

### View the first 20 lines of system logs
```bash
head -20 /var/log/syslog
```
**Purpose:** Displays the beginning of the system log file.

### Search for specific keywords inside logs
```bash
grep "error" /var/log/syslog
```
**Purpose:** Searches the log file for entries containing the word **error**.

### Display login history
```bash
last
```
**Purpose:** Shows the login history of users.

### Display failed login attempts
```bash
lastb
```
**Purpose:** Displays unsuccessful login attempts.

### Show currently logged-in users
```bash
who
```
**Purpose:** Displays users who are currently logged into the system.
